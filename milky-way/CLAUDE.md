# milky-way Development Guide

Kubernetes infrastructure-as-code using Tanka and Jsonnet for the home lab clusters.

## Running Commands (One-liner Pattern)

Claude Code cannot enter interactive shells. Use the one-liner pattern:

```bash
# Simple command
nix develop ./flake-profiles/milky-way-devenv --impure -c <command>

# Command that needs to run from milky-way/
nix develop ./flake-profiles/milky-way-devenv --impure -c bash -c 'cd milky-way && <command>'
```

## Tanka Commands

Preview, diff, and apply Kubernetes manifests:

```bash
# Preview generated manifests
nix develop ./flake-profiles/milky-way-devenv --impure -c bash -c 'cd milky-way && tk show environments/stage00/orion-system'

# Show diff before applying
nix develop ./flake-profiles/milky-way-devenv --impure -c bash -c 'cd milky-way && tk diff environments/stage00/orion-system'

# Apply changes (auto-approve for non-interactive)
nix develop ./flake-profiles/milky-way-devenv --impure -c bash -c 'cd milky-way && tk apply environments/stage00/orion-system --auto-approve=always'
```

> **Gotcha: cilium config changes don't restart the cilium agents.** `tk apply` to
> cilium Helm values (`charts.jsonnet`) updates the `cilium-config` ConfigMap but does
> **not** roll the cilium DaemonSet — the vendored chart adds no config-checksum
> annotation, so the running agents keep the old config (and `kubectl rollout status
> ds/cilium` reports success because the pod template is unchanged). After such a change:
> ```bash
> kubectl --context methanol -n kube-system rollout restart ds/cilium
> # confirm it took effect, e.g.:
> kubectl --context methanol -n kube-system exec ds/cilium -c cilium-agent -- \
>   cilium-dbg status --verbose | grep -iE "Socket LB|KubeProxyReplacement"
> ```

## Remote Commands on Methanol

### kubectl

Use the `methanol` context:

```bash
kubectl --context methanol get pods -A
kubectl --context methanol -n democratic-csi get pods
kubectl --context methanol -n democratic-csi logs -l app.kubernetes.io/name=democratic-csi --tail=100
```

### SSH Commands

Run commands on the storage server:

```bash
# Check ZFS volumes
ssh democratic-csi@methanol.local 'zfs list -r rpool/k8s/democratic-csi'

# Check ZFS properties (PVC metadata)
ssh democratic-csi@methanol.local 'zfs get -r k8s:pvc-ns-name rpool/k8s/democratic-csi/my-zfs-generic-iscsi'

# Check iSCSI targets
ssh democratic-csi@methanol.local 'sudo targetcli ls /iscsi'

# Clean up orphaned ZFS volume
ssh democratic-csi@methanol.local 'sudo zfs destroy rpool/k8s/democratic-csi/my-zfs-generic-iscsi/pvc-<uuid>'
```

## Directory Structure

```
milky-way/
├── environments/           # Tanka environments (spec.json + main.jsonnet)
│   └── stage00/orion-system/  # Methanol cluster config
├── charts/                 # Helm charts (vendored)
│   ├── democratic-csi/
│   ├── ingress-nginx/
│   └── kube-prometheus-stack/
├── charts.jsonnet          # Helm chart definitions with values
├── lib/                    # Reusable Jsonnet libraries
├── secrets/                # sops-nix symlinks (READ-ONLY)
└── vendor/                 # Jsonnet dependencies (tanka-util, etc.)
```

## Secrets Management

Secrets in `milky-way/secrets/` are **read-only symlinks** generated by sops-nix. To update secrets, modify the source in the sops-nix configuration and rebuild.

For development, inline literal values directly in config files when needed.

## Helm Charts

Charts are defined in `charts.jsonnet`:

```jsonnet
local helm = tanka.helm.new(std.thisFile);
{
  zfs_iscsi: helm.template("zfs-iscsi", "./charts/democratic-csi", {
    namespace: "democratic-csi",
    values: { ... },
  }),
}
```

Update chart dependencies:

```bash
nix develop ./flake-profiles/milky-way-devenv --impure -c bash -c 'cd milky-way && jb update'
```

## Debug Workflow Example

Full cycle for debugging democratic-csi:

```bash
# 1. Check current state
kubectl --context methanol -n democratic-csi get pods
kubectl --context methanol get pvc,pv -A

# 2. Check controller logs
kubectl --context methanol -n democratic-csi logs deployment/zfs-iscsi-democratic-csi-controller -c driver --tail=50

# 3. Make config changes and apply
nix develop ./flake-profiles/milky-way-devenv --impure -c bash -c 'cd milky-way && tk apply environments/stage00/orion-system --auto-approve=always'

# 4. Restart controller to pick up new config
kubectl --context methanol -n democratic-csi rollout restart deployment zfs-iscsi-democratic-csi-controller

# 5. Verify on storage server
ssh democratic-csi@methanol.local 'zfs list -r rpool/k8s/democratic-csi'
```

## Test Resources

Create a test StatefulSet with iSCSI storage:

```bash
# Check test pod
kubectl --context methanol -n test-k8s get pods,pvc

# Delete and recreate test resources
kubectl --context methanol -n test-k8s delete statefulset iscsi-test
kubectl --context methanol -n test-k8s delete pvc storage-iscsi-test-0
# Then reapply with tanka
```
